# Design docs

This directory holds the versioned design history of pip. Every significant architectural decision or forward-looking RFC lands here as a dated markdown file, reviewed via PR like the rest of the code.

## Conventions

- **Filename:** `YYYY-MM-DD-short-topic-design.md`
- **Status** at the top of each doc: `Draft` → `Accepted` → `Superseded` / `Deprecated`
- **One design per file.** If a decision is revised, write a new doc that supersedes the old one rather than editing history in place.
- **Link, don't duplicate.** Other docs (`README.md`, `docs/getting-started.md`, `docs/api/*`) should link here for the *why*, not restate it.
